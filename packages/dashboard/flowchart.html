Section C
AI Workers
Three concurrent workers using different LLM providers to maximize throughput
In plain English

Three AI workers chew through the queue simultaneously, each using a different AI provider so they don't compete for the same rate limits. If one provider throttles a worker, it parks the listing back in the queue and waits — the other two keep running.

Section D
M2 Pipeline — The Assembly Line
Every listing passes through 7 sequential processing steps
In plain English

Every listing goes through a 7-step assembly line. By the end, we know the full story of the car and how trustworthy the listing is. A perfect listing scores 100. Every missing field costs 8 points. Listings below 70 are rejected automatically; 70–89 go to a human review queue.

STEP 1
Extract
Read the listing. Pull out make, model, year, price, mileage, transmission, fuel type. Kijiji: instant structured data. Facebook: AI reads the free-text description.
STEP 2
Validate
Fix naming inconsistencies. "VW" → "Volkswagen". "2-door" → standardized body type. Verify the year is realistic (2000–2026).
STEP 3
Vision
Photo analysis via AI vision model. Guess the exterior colour from the listing's main photo. Fills a field sellers often forget.
STEP 4
CARFAX
Look up VIN in CARFAX database. Returns accident history, number of previous owners, open liens, and service records if available.
STEP 5
Redact PII
Strip seller's phone number and email address from the description text. Keeps our platform clean and protects users.
STEP 6
Score
Rate listing quality 0–100. Full marks for complete, clean data. Deduct 8 points per missing required field. Flag anomalies.
STEP 7
Save
Write the enriched listing to Neon Postgres. Trigger email alerts for any matching saved searches. Route to Published, Review, or Rejected.

Section E

Section F
Dashboard & Routes
Express.js server on port 3030 — deployed live on Railway
In plain English

One Express.js server serves two audiences: the team's operations dashboard (pipeline stats, reviewing flagged listings, triggering re-runs) and the public-facing car search website. It's deployed live on Railway so it's always accessible.

⚙ Admin — Operations Dashboard
GET /
Pipeline overview — scraper status, queue depth, listings ingested today, error counts
GET /review
Review queue — browse listings scoring 70–89, approve or reject them manually
POST /approve/:id
Approve a flagged listing — moves it to published state
POST /reject/:id
Reject a flagged listing — removes from public view
POST /rerun-ai
Bulk re-process listings through M2 Pipeline — useful after model upgrades
GET /stats
JSON endpoint — pipeline metrics, score distributions, source breakdown
Car Buyers — Public Website
GET /home
Homepage — hero search bar, browse by make/body type, featured listings
GET /browse
CarGurus-style results — filter by make, model, year, price, mileage, colour, source
GET /listing/:id
Full listing detail — all enriched fields, CARFAX report, photos, original source link
POST /alerts
Subscribe to email alerts — enter email + search criteria, get notified on match
GET /api/listings
JSON API — returns paginated, filtered listings for frontend rendering
Live deployment
Hosted on Railway — automatic deploys from GitHub main branch. Always-on, no cold starts.
